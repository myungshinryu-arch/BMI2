1. 통합 포털 홈 사용법

통합 포털 홈은 BM-Intelligence의 주요 분석 모듈로 이동하고, 통합 검색과 AI 기능을 실행하는 시작 화면입니다.

메인 화면 핵심 구성 및 사용 방법

1

통합 검색 기능 활용

• 설명: 화면 중앙의 검색창을 통해 타이어명, 제조사, 브랜드, 세그먼트, 컴파운드 원료, PLC 등 벤치마킹 관련 핵심 키워드를 기입해 탐색할 수 있습니다.
• 조작법:
1. 검색 입력창을 선택한 후 키워드(예: Michelin, Silica)를 1글자 이상 입력합니다.
2. 입력 조건에 대응하는 모듈 정보와 검색 결과 목록이 드롭다운 형태로 하단에 실시간으로 표시됩니다.
3. 검색 목록에서 원하는 항목을 클릭하면 해당 분석 페이지의 상세 영역으로 즉시 연동되어 이동합니다.
4. 검색어가 명확하지 않은 경우에는 관련성이 높은 분석 모듈을 시스템에서 우선하여 추천해 줍니다.
5. 우측의 [X] 버튼을 누르면 입력한 텍스트가 모두 제거됩니다.

2

헤더 내 주요 통합 연동 버튼 활용

• 설명: 우측 상단에 배치된 연동용 바로가기 버튼들을 통해 신속하게 특정 핵심 기능들을 수행할 수 있습니다.
• 기능별 정의:
- [통합 Report 작성]: 현재 화면 데이터와 분석 context를 바탕으로 임원 보고용 리포트를 실시간으로 생성하는 페이지로 이동합니다.
- [AI에게 질문하기]: 내부 데이터, BI News, Web Grounding을 유기적으로 교차 활용하는 Hybrid AI Agent와 실시간 질의응답을 수행할 수 있는 대화방으로 이동합니다.

3

핵심 분석 모듈 바로가기 링크 제어

• 설명: 통합 검색 패널 하단에 가로로 나열된 5대 핵심 분석 카드 영역입니다.
• 사용법: 시장/상품전략, Tire BM Report, Comp'd BM Report, 경쟁사 BI News, AI Simulator 등 카드 내 수록된 개별 메뉴 링크(예: [컴파운드 탐색기], [전체 뉴스 피드] 등)를 선택하면 해당 분석 서브 탭이 바로 활성화된 상태로 즉시 연결됩니다.

4

Global Top4 제조사 동향 대시보드 및 연혁 팝업

• 사용법: 포털 홈 하단의 글로벌 매출 및 판매량 동향 그래프 영역에서 필터(연도, 세그먼트)를 변경하여 차트 정보를 실시간으로 관제할 수 있습니다. 우측 상단의 [History] 텍스트를 선택하면 시스템의 동기화 연혁 모달창이 활성화되며, [X] 버튼으로 언제든 닫을 수 있습니다.

2. 시장/상품전략 사용법

글로벌 Top4 제조사의 세그먼트별 매출, 판매량 정보 및 점유율 비중, 대표 타이어 세대별 성능 진화 궤적을 비교하여 중장기 로드맵을 확인하는 전략 가이드입니다.

주요 탭 및 기능별 상세 조작 방법

1

[Market & Sales Overview] 탭 조작

• 기능 및 위치: 글로벌 거시 동향 및 주요 4대 제조사의 점유율을 확인하는 화면입니다.
• 조작법:
1. 상단의 필터 영역에서 분석 연도(Year)와 세그먼트 드롭다운을 설정합니다.
2. 하위 권역별 선택 탭(예: [북미 시장], [유럽 시장])을 상호 클릭하여 타깃 시장을 지정합니다.
3. 조건 변경에 맞춰 제조사별 매출 점유율 비중 및 연도별 판매량 추이 곡선 그래프가 실시간으로 재구성되는 것을 확인합니다.

2

[기술 경쟁력 Overview] 탭 조작

• 기능 및 위치: 제조사 간의 성능 평점 지표를 레이더 차트로 교차 탐색하는 영역입니다.
• 조작법:
1. 화면 내 거대한 6대 성능 분석 레이더 차트에서 상단의 범례(예: Michelin, Continental)를 마우스로 클릭하여 특정 선을 표시하거나 숨깁니다.
2. 성능 꼭짓점 좌표 영역에 마우스를 호버하면 자사 타이어와 경쟁 모델 간의 구체적 타깃 평점 지수 격차를 툴팁 수치로 즉시 확인할 수 있습니다.

3

[대표 상품 세대별 성능] 탭 조작

• 기능 및 위치: 주요 브랜드 타이어의 역사적 세대교체에 따르는 성능 변화 추이를 분석하는 영역입니다.
• 조작법:
1. 테이블 상에서 세대별(1세대 ➡️ 2세대 ➡️ 3세대) 진화 항목을 정렬 대조합니다.
2. 점탄성 핵심 지표 및 마모 효율 변화 추이에 대한 시각화 그래프를 확인하여 경쟁사 기술 발달 방향성을 확인합니다.

4

[경쟁사 R&D 전략] 및 [대응 전략 & 로드맵] 탭 조작

• 기능 및 위치: 당사의 2026~2030 중장기 실행 로드맵 및 기술 보강 대안을 확인하는 장표입니다.
• 조작법:
1. [대응 전략 & 로드맵] 내부 탭을 선택한 후 마우스 스크롤이나 가로 슬라이드를 통하여 시기별 대응 맵을 검토합니다.
2. 상단의 [단기 보강 과제], [중기 개발 과제] 등의 컬러 뱃지를 선택하면 연동되는 실행 기술의 요약 상세 위치로 화면 포커스가 자동으로 미끄러져 정렬됩니다.

3. Tire BM Report 사용법

경쟁사 완성 타이어의 실측 성능 사양, PLC(제품수명주기) 타임라인 맵, EV 특화 친환경 타이어 필터 정보 및 사내 보고서 아카이브를 활용하는 조작법을 다룹니다.

4대 핵심 하위메뉴 및 실무 조작 절차

1

[종합 분석 대시보드] 탭 대형 테이블 정렬 및 비교

• 사용법:
1. 타이어 정보가 나열된 대형 테이블의 헤더 컬럼명(예: [Groove Depth], [Pitch Count], [UTQG])을 선택합니다.
2. 1회 클릭 시 해당 항목 기준 내림차순(높은 순), 재클릭 시 오름차순(낮은 순)으로 리스트가 자동 정렬되어, 자사와 비교 제조사 모델 간의 정량적 사양 차이를 직관적으로 파악할 수 있습니다.

2

[PLC Timeline Map] 탭 조작

• 사용법:
1. 브랜드별 타이어 제품들의 출시부터 단종 주기를 다이어그램으로 제공합니다.
2. 마우스 휠 조작 또는 마우스를 좌우로 드래그하여 과거(2021년)부터 미래(2028년 이후)로 이어지는 글로벌 신제품 출시 연혁 블록을 연도별로 탐색합니다.

3

[EV 친환경 특화 보드] 탭 다차원 필터링

• 사용법: 전기차 타이어에 중점을 둔 비교 탭입니다. 고하중 지수, 전기차 전용 흡음재 장착 여부, 회전저항 1등급 적용 여부 등을 검증하는 전용 토글 뱃지를 각각 클릭함으로써 자사 iON 브랜드와 경쟁사 대표 친환경 라인업 간의 수치를 선택적으로 발췌해 비교합니다.

4

[보고서 라이브러리] 탭 PDF 아카이빙 다운로드

• 사용법: 타이어 BM 정밀 벤치마킹 완료 자료 리스트 중 필요한 행을 선택한 뒤, 우측 다운로드 및 뷰어 아이콘을 클릭하여 내장된 PDF 보고서 원본을 로컬 시스템에 저장하거나 확인할 수 있습니다.

4. Comp'd BM Report 사용법

고무 컴파운드의 실측 물성 빅데이터를 비교 분석하고, 화학적 구성 성분 및 정량 엑셀 데이터를 바탕으로 메이커별 배합 설계 차이를 규명하는 분석기입니다.

4대 핵심 하위메뉴별 세부 가이드

1

[제품 성능 비교] 탭 및 동적 대시보드 필터 활용

• 사용 가이드 (Premium v1.5 기준):
1. 화면 상단의 **[Season]** 및 **[Segment / 차종]** 드롭다운 필터를 선택하면 조건에 매칭되는 데이터셋으로 전체 카드가 재정렬됩니다. (사용 편의성을 위해 필터 드롭다운과 텍스트 크기가 52px 높이, 1.15rem으로 상향되어 시인성이 우수합니다.)
2. 글로벌 타이어 트렌드를 반영하여 기존 메이커 외에 **9번째 독립 제조사로 금호타이어(KUMHO)**가 공식적으로 추가 정렬되어 있으며, 각 메이커는 개별 카드 형태로 구분 제공됩니다.
3. 불필요한 선택 가이드 문구 없이 각 브랜드 타이틀(1.95rem)과 패턴명, 샘플 수량, 가교밀도 텍스트가 대폭 확대 적용되어 한눈에 명확히 들어옵니다.
4. 폴리머 블렌드 분석 내에서 **합성고무(SBR) 고유 컬러가 파란색(Blue, #2563eb)으로 지정**되어 타 원료(NR, BR)와의 구분이 용이합니다.
5. 마우스 커서로 점탄성 온도 스윕 차트 상단의 개별 범례명을 클릭하여 특정 곡선만을 실시간 온/오프 필터링할 수 있습니다.

2

[컴파운드 탐색기] 탭 조작

• 사용법:
1. 데이터 소스 드롭다운 메뉴를 좌클릭합니다.
2. 리스트에서 **[트레드 컴파운드]**, **[케이스 컴파운드]**, **[TBR 벤치마킹]** 중 원하는 영역을 선택합니다.
3. 선택 즉시 하단의 원천 성분 함량 표 및 실리카Phr, 가교밀도 비교 테이블이 해당 데이터베이스 영역으로 자동 교체됩니다.

3

[BM Report 생성기] 탭 개편 적용 가이드

• 사용법:
1. 제조사 선택 및 구체적 타이어 모델 선택 드롭다운을 설정합니다.
2. **[BM Report 생성]** 버튼을 선택하면 고무 컴파운드의 정밀 물성 리포트가 드로잉됩니다.
3. 리포트는 좌측 주황색 바가 장착된 2대 대형 분석 섹션으로 체계화되어 있습니다:
- **[섹션 1] Comp'd 물성 분석**: 레이더 차트, ARES 곡선 및 물성 상관관계 분석 시뮬레이터 수평 통합 연동.
- **[섹션 2] Comp'd 배합제 분석**: Polymer Blend 비율, Styrene/Vinyl 비율, 보강제, 가류제/촉진제 정량 분석 차트가 하나의 박스로 깔끔하게 통합 그룹화되어 있습니다.

4

[Maker 상품 비교] 탭 오버레이 제어

• 사용법: 다자간 밸런스 레이더 차트 탭입니다. 성능을 대조할 복수의 메이커 체크박스를 활성화하면, 꼭짓점이 중첩된 기하학적 차트 영역이 오버레이 출력되어 각사 제품 성능 방향성의 전반적인 팽창/수축 격차를 시각적으로 직접 판단할 수 있습니다.

Menu 05

5. 경쟁사 BI News 사용법

실시간 수집된 글로벌 메이커(Michelin, Continental, Bridgestone, Goodyear 등)의 최신 특허, R&D 기술 뉴스 및 감성 분석을 확인하고 AI 기반의 핵심 요약을 활용하는 조작법입니다.

3대 핵심 서브 메뉴별 조작 방법

1

[전체 뉴스 피드] 탭 및 브랜드 필터링

• 사용법:
1. 화면 상단에 정렬된 제조사 필터 단추(예: [Michelin], [Continental] 등)를 개별 선택하여 특정 제조사의 뉴스만을 분류 및 필터링할 수 있습니다.
2. 기사 카드 항목을 좌클릭하면 가독성이 확보된 상세 정보 팝업창이 표출되어 기사 원문 및 수집 메타데이터를 정독할 수 있으며, 우측 상단 [X] 버튼을 누르면 팝업이 소거됩니다.

2

[제조사 트렌드 분석] 탭 차트 탐색

• 사용법: 자연어 처리를 통해 산정된 메이커별 뉴스 긍정/부정 비율과 월간 뉴스 노출 빈도를 차트화하여 보여줍니다. 그래프의 막대 기둥이나 데이터 포인트 상에 마우스 커서를 올리면 월별 기사 수량 및 백분율 지수가 실시간 툴팁으로 반환되어 정확한 통계를 확인할 수 있습니다.

3

[AI 핵심 동향 요약] 브리핑 활용

• 사용법: 수집된 벌크 데이터를 바탕으로 AI가 요약 분석한 3개 카드를 정독할 수 있습니다. 각 카드는 [1. 핵심 기술 동향 요약], [2. 자사 비즈니스 위협 영향], [3. 당사 대응 전략 권고안] 등 기획 업무 및 전략 수립에 필요한 알짜 정보들로 체계화되어 있습니다.

6. AI Simulator 사용법

벤치마크 타이어의 고무 원재료 슬라이더를 마우스로 직접 조절하여 가상 레시피를 가공 설계하고, AI가 예측 연산한 점탄성 차트 및 6대 성질 지수를 실시간 비교 추적하는 엔지니어링 패널입니다.

가상 배합 설계 및 시뮬레이션 상세 프로시저

1

벤치마크 타이어 선택

• 조작법: 화면 좌측 상단의 벤치마크 드롭다운 메뉴를 선택하여 대조 대상이 될 기준 경쟁사 모델(예: Michelin PS5 등)을 지정합니다. 지정 시 해당 타이어의 실제 계측 물리 정보가 초기 기준값으로 로드됩니다.

2

Virtual 레시피 디자인 (원료 함량 슬라이더 튜닝)

• 조작법:
1. 좌측 배합 제어부의 **Silica (phr)**, **TESPT (%)**, **Oil (phr)** 등 함량 조절 슬라이더 노브를 좌우로 드래그하여 가상의 신규 배합안(Virtual Recipe)을 실시간으로 설계합니다.
2. 더 정밀하게 소수점 단위 미세 조정을 수행하고 싶을 때에는, 슬라이더 노브를 1회 마우스 클릭하여 선택한 후 키보드 좌우 방향키(←, ➡️)를 누르면 0.1 phr / 0.1% 간격으로 수치 미세 튜닝이 구동됩니다.

3

ARES tanδ Temperature Sweep 분석

• 사용법: 배합 슬라이더의 변경 위치에 연동되어 우측 차트 내 가상 예측선(점선 형태)이 실시간으로 갱신 렌더링됩니다. 온도별 tanδ 곡선의 변동 거동(예: 0℃ 회전저항 특성, 60℃ 연비/제동 밸런스 등)을 눈으로 면밀하게 추적 비교할 수 있습니다.

4

성능 비교 분석 보드 (정량 지표 점수 대조)

• 사용법:
1. 화면 우측 영역의 6대 성능 밸런스 레이더 차트와 상세 스코어 보드를 확인합니다.
2. 사용자가 설계한 Virtual 배합과 실제 Benchmark 타이어 간의 **Wet 제동 성능, 연비(RR), 내마모 점수** 등 주요 6성 지수 격차를 100점 만점 평점으로 대조 관제하여 배합 최적 지점을 도출해 냅니다.

5

AI 레시피 개선 가이드 실행

• 사용법: 가상 배합 튜닝 도중 성능을 더욱 비약적으로 극복하기 위해, 제어판 하단의 **[AI 레시피 개선 가이드]** 버튼을 선택합니다. 인공지능이 내장 데이터 기반으로 현 슬라이더 조건의 임계점을 분석하여 우수한 목표 성능 평점에 도달할 수 있는 추천 개선 후보 배합안 리스트를 연출해 줍니다.

6

추천 배합 적용 및 반영

• 사용법: 개선 가이드 팝업창에서 추천된 AI 후보 배합 조건들 중 적합한 설계안을 선택하고 **[배합 적용]** 버튼을 클릭합니다. 즉시 해당 가이드 배합 수치가 좌측 가상 슬라이더 노브 위치로 정합 자동 세팅되며 차트 분석값이 즉각 일체 재반영됩니다.

7. 통합 Report 작성법

현재 사용자의 분석 맥락(context)과 데이터베이스 실측값들을 종합적으로 취합하여, LLM 엔진을 통해 비즈니스 의사결정용 하이엔드 보고서 장표를 자동 작성하고 출력하는 기능 가이드입니다.

임원 보고용 지능형 리포트 빌드 프로세스

1

데이터 연동 조건 설정

• 사용법: 보고서 생성기 페이지 상단의 타깃 제조사 및 대상 타이어 제품군 명칭을 드롭다운에서 설정해 지정합니다. 시스템은 선택값에 최적화하여 1:1 대응하는 자사(한국타이어)의 매칭 타이어 데이터 정보를 로딩할 준비 상태를 마칩니다.

2

지능형 리포트 자동 생성 단추 클릭

• 사용법:
1. 드롭다운 지정 후 우측의 **[BM Report 생성]** 단추를 누릅니다.
2. Gemini LLM 엔진이 현재 대시보드 맥락과 내부 실측 데이터들을 완벽히 정적/동적으로 수집 파악하여 실무 6대 보고서 장표 영역에 데이터를 안전하게 로드합니다.
3. 수치 정보와 명칭들은 모두 실제 보관 데이터에 기반하여 정확하고 일관되게 채워지며, 임원 보고 목적에 알맞은 핵심 요약 브리핑 및 권고 추진 과제 등의 시사점 문안이 자동으로 정밀하게 렌더링됩니다.

3

Fallback 안정성 탑재

• 설명: 만일의 외부 네트워크 일시 단절, Cloud Run 또는 Vertex AI 호출 장애 등으로 인해 실시간 LLM 답변 전송에 지연이나 에러가 유발될 경우, 사전에 로컬에 세팅된 고품질 고정 데이터셋 기반의 Local Fallback 분석 시나리오 데이터로 즉시 교체 대체되어 보고서 레이아웃의 안전성이 유지됩니다.

4

보고서 인쇄 및 PDF 저장 조작 (매우 중요)

• 조작법:
1. 화면 최상단 우측의 **[Report 인쇄 및 PDF 저장]** 단추를 선택합니다.
2. 표준 웹브라우저 인쇄 설정 가이드 팝업창이 기동되면 다음과 같이 설정값을 변경해 주어야 완벽히 출력됩니다:
- 인쇄 용지 방향: 반드시 **[가로(Landscape)]**로 변경 선택합니다.
- 설정 더보기 옵션: **[배경 그래픽(Background Graphics)] 항목을 마우스로 체크해 켭니다.** (이 옵션이 활성화되지 않으면 보고서의 미려한 카드 배경색 그라데이션 및 그리드 선이 출력에서 누락될 수 있으므로 사전에 필히 체크해 주십시오.)
3. 미리보기에서 다크 테마 인포그래픽 정렬 정합성을 최종 확인하고 인쇄 또는 PDF 저장 버튼을 선택합니다.

8. AI에게 질문하기 사용법

내부 지식 데이터베이스(RAG), 실시간 수집된 최신 경쟁사 기술 뉴스, Web Grounding 기술을 유기적으로 하이브리드 교차 구동하여 정교한 자연어 답변을 반환하는 AI Insight Agent 대화방입니다.

Hybrid AI Agent 전송 및 식별 뱃지 판독법

1

질문 유형에 따른 지능형 하이브리드 라우팅 자동 연동

• 조작법: 챗방 하단 텍스트 창에 타이어 성능, 컴파운드 설계, 경쟁사 최근 출시 일정, R&D 트렌드 등에 관한 궁금한 사항을 자유롭게 자연어로 입력하고 전송 버튼을 누릅니다.
• 라우팅 작동 구조:
- **내부 데이터 질문**: 대시보드 물성 정보나 내부 저장 자료(RAG context)를 최우선 매핑하여 팩트 중심의 정량 수치로 정밀하게 답변합니다.
- **최신 출시/뉴스/동향 질문**: 내부 데이터에 수록되지 않은 최신성/외부 시장 정보는 실시간 구글 웹 검색을 가동해 Web Grounding 기반의 가장 공신력 있는 내용으로 발췌하여 기술해 줍니다.
- **하이브리드 결합 질문**: 자사 수치와 외부 최신 동향을 통합 대조하는 질문의 경우, 내부 수치 데이터셋과 실시간 외부 웹 기사 정보를 일련적으로 조합해 하이브리드 분석으로 명쾌히 풀어내 줍니다.
- **시뮬레이터 관련 질문**: 가상 배합 설계나 tanδ 분석 시, 현재 튜닝 중인 내부 Simulator context 값을 우선 활용하여 대답을 완성합니다.

2

답변 근거 구분용 5대 시스템 식별 뱃지 확인 요령

• 설명: 대화형 버블 상단 또는 우측 영역에 출력되는 컬러풀한 시스템 전용 뱃지를 활용해 인공지능이 생성한 답변의 원천 근거를 한눈에 신뢰 및 검증할 수 있습니다.

Gemini LLM : Google Vertex AI의 최상위 Gemini LLM 추론 모델을 사용했음을 인증하는 표식입니다.
내부 데이터 기반 : 답변이 외부 추정이 아니라 엄선된 사내 벤치마킹 데이터베이스(RAG) 원천 수치를 참조해 구성되었음을 보증합니다.
Web Grounding 사용 : 실시간 웹서치 기술(Google Search Grounding)과 교차 검증을 수반하여 도출된 신뢰할 만한 외부 팩트 정보임을 뜻합니다.
Hybrid 분석 : 내부 정밀 물성 수치와 외부 시장 트레이드 동향 기사를 하나의 논리로 동적 결합했음을 표기합니다.
Local Fallback : 네트워크 정체나 API 에러 유발 시 오답을 임의 추론(Hallucination)하는 대신, 로컬에 저장 보관된 기본 지식 시나리오로 대체하여 정답을 안전하게 대체 표출한 상태를 나타냅니다.

9. FAQ 및 오류 대처

플랫폼 주요 모듈을 이용하시다 예기치 못한 작동 정지나 데이터 누락 현상 마주 시 현업 연구원이 신속히 조치할 수 있는 트러블슈팅 매뉴얼입니다.

Q1

Q. 분석 화면이나 대시보드 내 특정 영역에 그래프나 정량 수치 표 데이터가 전혀 노출되지 않습니다.

• 원인: 데이터베이스 API 커넥터 모듈의 순시적 응답 지연 또는 로컬 웹브라우저 리소스 로딩 캐시 꼬임이 주 원인입니다.
• 해결 조치 단계:
1. 사용 중인 브라우저 화면에서 **[Ctrl + F5]** 키를 동시에 입력하여 강력하게 강제 캐시 초기화 및 새로고침을 실행해 줍니다.
2. 새로고침 후에도 문제가 지속되면, 로컬 네트워크의 연결 상태와 시스템 백엔드 API 서버 포트 통신망이 가동 중인지 IT 부서나 인프라 관리자에게 연결 상태 점검을 조율하십시오.

Q2

Q. AI에게 질문하기 및 통합 Report 생성 도중 답변 근거 뱃지에 'Local Fallback' 상태만 표시됩니다.

• 원인: 백엔드 uvicorn API 서비스 장애, Vertex AI Cloud 인증 계정 토큰 만료, 또는 허용 IP 이탈 등이 원인입니다.
• 해결 조치 단계:
1. FastAPI 서버 엔진(`app:app`)이 백그라운드에서 정상 가동되고 있는지 CLI 명령창 상태를 진입 조회합니다.
2. Cloud Run API 주소 상태 및 Vertex AI 프로젝트(`hk-hackathon-01`), 리전(`global`)에 대한 로컬 ADC(Application Default Credentials) 권한 자격 증명 활성화 상태를 면밀히 점검하십시오.

Q3

Q. 메인 통합 검색창에 검색어를 기입했는데 결과 없음 상태만 표출되어 이동이 불가능합니다.

• 원인: 오탈자가 있거나 너무 축약된 기호를 입력했을 경우 시스템 매핑 범위를 초과하게 됩니다.
• 해결 조치 단계:
1. 입력어를 조금 더 대중적인 제조사명(예: Michelin, Kumho), 모델명(예: PS5, Ventus), 세그먼트명, 혹은 특정 고무 원재료 키워드로 풀네임을 더 구체적으로 정밀하게 기입해 보시면 정상 추천 필터가 즉각 되살아납니다.

Q4

Q. AI Simulator에서 원재료 함량 슬라이더 바를 분명 이동했는데도 오른쪽 tanδ 예측 그래프가 변동하지 않고 정지해 있습니다.

• 원인: 벤치마크 대상 타이어 모델이 정상적으로 체크 지정되지 않았거나, 브라우저 스크립트 실행 스레드의 일시 지연 상태입니다.
• 해결 조치 단계:
1. 좌측 최상단의 대조 벤치마크 타이어 드롭다운 선택 메뉴에서 정확한 비교 모델이 유효하게 로딩 지정되어 있는지 다시 한 번 명확하게 체크 기입하십시오.
2. 슬라이더 노브를 살짝 클릭하여 마우스 포커스를 준 뒤, 키보드 방향키를 톡톡 쳐서 입력 상태 이벤트를 강제로 재호출하시면 그래프 선이 실시간으로 부드럽게 다시 동기화 갱신됩니다.
